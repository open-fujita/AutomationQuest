# requirements — DS 学習シミュレーションゲーム（縦切り初版）

> 親 INDEX: [[INDEX]]

## 背景・目的
BizRobo! Design Studio は独特の概念（ステップ / ブランチ / タイプ / ループ / トライ-キャッチ）と用語の壁が高い。
DS 画面を忠実に再現した Web アプリ上でゲームを進めるうちに、自然とロボットを作れるようになる学習体験を提供する。
ゲームの主軸は「自動化思考＝推理」。対象は Design Studio のみ（MC / RS は対象外）。

## ユーザーストーリー
- 初学者として、手作業の現場を観察し「何が繰り返しか/変動するか/例外か」を推理することで、自動化の発想を身につけたい。
- 初学者として、本物の DS 画面（マイプロジェクト / ロボットビュー / ブラウザビュー / データの状態 / ステップビュー / ステータスビュー）で操作し、用語と操作を体で覚えたい。
- 初学者として、組んだロボットを実行し「手作業◯分 → ロボット◯秒」の効果が見えることで達成感を得たい。
- 学習者として、間違えたとき「何が足りないか」を的確に指摘してほしい。

## 受け入れ条件（初版）
1. `npm run dev` で DS 風 4 領域 UI + ツールバーが表示される。
2. Mission 1（新規ロボット → ページを読み込む → 単一抽出 → 実行 → 効果測定）が通しで遊べる。
3. Mission 2（タイプ/変数 → 要素の繰り返しで全件抽出 → 二重登録の推理オチ）が通しで遊べる。
4. 誤操作（抽出未設定 / ループ抜け 等）に対して受け入れ条件チェックが的確なヒントを返す。
   - 特に「ループが無いと先頭 1 件しか取れない」を体験できる。
5. 進捗（クリア状況 / 推理回答 / 用語解禁）が localStorage に保存され、リロード後も保持される。
6. BizRobo 正規用語が UI・用語集に自然に登場する。
7. `npm run build` と `tsc --noEmit` が通る。

## 制約
- React + TypeScript + Vite / React Flow / Tailwind / Zustand。バックエンド無し（純クライアント）。
- 依存はプロジェクトローカルのみ（`npm install -g` 禁止）。
- 実サイトにはアクセスしない（模擬サイトは定義データ）。
- ロボットモデル・健康度スコアの命名は bizrobo-analyzer-byDevin の SCHEMA.md に準拠。
- 画像は gpt-image-2 で生成し、最小枚数に絞る。UI 枠は CSS で描く。
